# Railway Deploy Checklist

## 1. Repository
- Ensure these files are committed:
  - `bucket.py`
  - `requirements.txt`
  - `Procfile`
  - `.gitignore`
  - `.env.example`
- Ensure `.env` is NOT committed.

## 2. Create Railway Service
- In Railway: New Project -> Deploy from GitHub Repo.
- Select this repository.
- Service should detect Python and use your `Procfile` command.

## 3. Runtime Command
- Confirm start command is:
  - `uvicorn bucket:app --host 0.0.0.0 --port $PORT`

## 4. Environment Variables
Set these in Railway service Variables (copy from `.env.example`):
- `SECRET_KEY`
- `COOKIE_SECURE=true`
- `APP_BASE_URL=https://<your-railway-domain>`
- `SUPERUSER_EMAIL=joaoccoliveira@live.com`
- `DEFAULT_SUBDIRECTORY=general`
- `INITIAL_API_TOKENS=<bootstrap-token-1>,<bootstrap-token-2>`
- `R2_BUCKET_NAME`
- `R2_ACCESS_KEY_ID`
- `R2_SECRET_ACCESS_KEY`
- `R2_ENDPOINT_URL` (or set `R2_ACCOUNT_ID`)
- Optional: `R2_PREFIX`, `SESSION_COOKIE_NAME`, `SESSION_MAX_AGE_SECONDS`

## 5. Database (Recommended)
- Add Railway Postgres plugin.
- Confirm `DATABASE_URL` appears in service variables.
- Do not rely on local SQLite for production persistence.

## 6. Public Access
- Enable public networking/domain in Railway.
- Open the generated domain.

## 7. Smoke Tests
- `GET /healthz` returns `{\"ok\": true}`.
- Register account using one of the values from `INITIAL_API_TOKENS`.
- Login works.
- Superuser account can create subdirectories.
- Non-superuser accounts can select and use all subdirectories.
- Upload file works.
- Download file works.
- Delete file works.

## 8. Security
- Rotate any previously shared keys/secrets.
- Keep `.env` local only.
- Use long random `SECRET_KEY`.

## 9. Common Failures
- Upload says R2 env vars missing:
  - Verify variables are set in Railway service (not only local `.env`).
  - Redeploy/restart service after changing vars.
- Registration says API token is invalid:
  - Confirm token exists in `INITIAL_API_TOKENS` or was generated by superuser.
  - Ensure token has not expired and has not already been used.
- Users disappear after redeploy:
  - You are likely on SQLite; move to Postgres.
